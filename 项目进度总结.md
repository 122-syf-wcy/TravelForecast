# 游韵华章 · 智慧文旅平台 — 项目进度总结

> **项目名称**：游韵华章 · 智游六盘水 (TravelForecast)
> **最后更新**：2026-03-03
> **项目定位**：集"AI数字伴游 + 沉浸式导览 + 红色研学 + 客流预测 + 文创电商"于一体的智慧文旅平台
> **目标景区**：贵州六盘水（梅花山、玉舍森林公园、乌蒙大草原、水城古镇、明湖湿地公园）

---

## 一、整体架构概览

项目采用**微服务架构**，包含以下子系统：

| 序号 | 子系统 | 技术栈 | 端口 | 状态 |
|------|--------|--------|------|------|
| 1 | **TravelForecastBackend** (主业务后端) | Spring Boot 3.2 + MyBatis-Plus + MySQL | 8080 | ✅ 已完成核心功能 |
| 2 | **TravelForecastFrontend** (Web前端) | Vue 3 + TypeScript + Element Plus + TailwindCSS + ECharts | 3000 | ✅ 已完成核心功能 |
| 3 | **TravelForecastingAIBackend** (AI智能服务) | Spring Boot 3.2 + 通义千问DashScope + Redis | 8081 | ✅ 已完成核心功能 |
| 4 | **TravelForecast-PythonPredictionService** (客流预测) | Python + FastAPI + ARIMA + LSTM | 8001 | ✅ 已完成核心功能 |
| 5 | **TravelForecast-DigitalHuman** (3D数字人) | Three.js + MapLibre + Python后端(TTS) | 8083 | ✅ 基本可用 |
| 6 | **TravelForecastGateway** (API网关) | Spring Cloud Gateway + Redis限流 + JWT | 8888 | ✅ 已完成 |
| 7 | **TravelForecastMiniProgram** (微信小程序) | Uni-app + Vue 3 | H5/微信 | ✅ 核心完成 |
| 8 | **TravelForecastMiniProgramBackend** (小程序后端) | Spring Boot + MyBatis-Plus | 8082 | ✅ 核心完成 |

## 九、给后续开发者/AI的注意事项

1. **角色体系**：系统有三种角色 `user`、`merchant`（商户端路由为 `/business`）、`admin`，存储在 `users.role` 字段中
2. **后端字段命名**：MyBatis-Plus自动将camelCase转为snake_case，注意实体字段与数据库列的对应关系
3. **景区图片**：`ScenicSpot.imageUrl` 经常为空，需从 `scenic_images` 表（`imageType=COVER`）获取封面图
4. **景区分类**：后端实体字段名为 `category`（非 `type`），前端映射时需注意
5. **状态值**：后端存储英文（`active`/`inactive`/`pending`），前端需翻译为中文显示
6. **缓存策略**：Landing页面API有10分钟缓存，管理端修改内容后需调用 `clearContentCache()` 清除
7. **文件上传**：统一使用阿里云OSS，通过 `/api/oss/upload` 或 `/api/upload` 接口
8. **AI服务**：需配置通义千问DashScope API Key才能使用AI功能
9. **预测服务**：Python服务独立运行，Java后端通过HTTP调用
10. **网关**：生产环境需修改JWT密钥、CORS白名单、限流参数
