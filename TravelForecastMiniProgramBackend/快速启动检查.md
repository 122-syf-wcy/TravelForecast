### 4. 端口检查
- [ ] 确认端口 8082 未被占用

## 🚀 启动步骤

### 方式一：IntelliJ IDEA
1. 打开项目：`TravelForecastMiniProgramBackend`
2. 找到主类：`com.travel.TravelMiniApplication`
3. 右键 → Run / Debug
4. 查看控制台输出，确认看到：
   ```
   Started TravelMiniApplication
   Tomcat started on port(s): 8082 (http)
   ```

### 方式二：命令行
```bash
cd F:\Web\TravelForecastMiniProgramBackend
mvn spring-boot:run
```

## 🔍 验证启动成功

### 1. 检查日志
启动成功后应该看到：
```
Started TravelMiniApplication in X.XXX seconds
Tomcat started on port(s): 8082 (http)
```

### 2. 测试接口
在浏览器或 Postman 中访问：
```
GET http://localhost:8082/api/auth/wechat/login
```
应该返回 405（Method Not Allowed），说明接口存在，只是需要 POST 方法。

### 3. 检查小程序前端
- 打开小程序开发者工具
- 进入"我的"页面
- 点击"点击登录"
- 查看控制台是否有请求发送到 `http://localhost:8082/api/auth/wechat/login`

## ❌ 常见问题

### 问题1：端口被占用
**错误信息**：`Port 8082 was already in use`

**解决**：
1. 检查是否有其他服务占用 8082
2. 修改 `application.yml` 中的端口号
3. 同时修改前端 `config.js` 中的 `MP_API_BASE_URL`

### 问题2：数据库连接失败
**错误信息**：`Communications link failure`

**解决**：
1. 确认 MySQL 服务已启动
2. 检查 `application.yml` 中的数据库连接信息
3. 确认数据库 `travel_forecast` 已创建

### 问题3：接口返回 404
**错误信息**：`POST http://localhost:8082/api/auth/wechat/login 404`

**解决**：
1. 确认后端服务已启动（检查控制台日志）
2. 确认 `WechatLoginController` 的路径是 `/auth/wechat/login`
3. 确认 `ApiPrefixConfig` 配置类已加载（添加了 `/api` 前缀）

### 问题4：跨域错误（CORS）
**错误信息**：`Access-Control-Allow-Origin`

**解决**：
- 已添加 `WebConfig` 配置类处理跨域
- 如果仍有问题，检查 `WebConfig` 是否被正确加载
