# 需求文档

## 简介

展示页（Landing页面）的轮播图图片无法正常显示，显示"FAILED"错误。经过分析发现：
1. 数据库中存储的是OSS完整URL（如：https://smarttourism717.oss-cn-beijing.aliyuncs.com/banners/202510/1_1_1760789864543.jpg）
2. 这些OSS URL可能因为权限问题或签名过期无法直接访问
3. 系统有ImageProxyController可以代理图片请求，但前端没有正确使用
4. 需要修复图片URL处理逻辑，确保轮播图能够正常展示

## 术语表

- **Landing页面**: 系统的首页展示页面，包含轮播图、功能介绍等内容
- **轮播图**: Banner图片，在页面顶部循环展示的图片
- **OSS**: 阿里云对象存储服务，用于存储图片文件
- **签名URL**: 带有访问权限签名的临时URL，用于访问私有OSS资源

## 需求

### 需求 1

**用户故事:** 作为访客，我想要在Landing页面看到正常显示的轮播图，以便了解系统的功能和特色。

#### 验收标准

1. WHEN 访客访问Landing页面 THEN 系统应当加载并显示所有启用的轮播图
2. WHEN 轮播图图片加载失败 THEN 系统应当显示占位图而不是"FAILED"错误
3. WHEN 后端返回轮播图数据 THEN 系统应当正确处理OSS签名URL和公共URL
4. WHEN 图片URL包含特殊字符 THEN 系统应当正确编码和解码URL
5. WHEN 图片不存在或无法访问 THEN 系统应当提供友好的降级方案

### 需求 2

**用户故事:** 作为管理员，我想要上传的轮播图能够正确存储和显示，以便维护系统的展示内容。

#### 验收标准

1. WHEN 管理员上传轮播图 THEN 系统应当生成正确的OSS对象名称
2. WHEN 系统存储轮播图URL THEN 系统应当使用统一的URL格式
3. WHEN 系统生成签名URL THEN 系统应当正确处理URL编码
4. WHEN 前端请求轮播图列表 THEN 后端应当返回可访问的图片URL

### 需求 3

**用户故事:** 作为开发者，我想要有清晰的图片URL处理逻辑，以便快速定位和修复图片加载问题。

#### 验收标准

1. WHEN 系统处理图片URL THEN 系统应当记录详细的日志信息
2. WHEN 图片加载失败 THEN 系统应当在控制台输出错误原因
3. WHEN 调试图片问题 THEN 开发者应当能够通过日志追踪URL转换过程
