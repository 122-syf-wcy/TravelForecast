### 复制以下指令发送给 AI：

**Role:** 你是一位精通 WebGL 和 GIS 开发的高级前端工程师。

**Goal:**
请帮我开发一个 **Web端 3D 数字人 Demo**。
我们需要在一个网页地图上加载一个 3D 虚拟人模型，并确保它能“站”在地图的特定坐标上，且随着地图的拖拽、缩放保持位置同步。

**Constraints (重要限制):**

1. **必须使用 MapLibre GL JS** 作为地图引擎（替代 Mapbox），以实现完全免费且无 Access Token 的部署。
2. **必须使用 Three.js** 处理 3D 模型渲染。
3. 输出为一个单独的 `index.html` 文件，包含所有必要的 CDN 链接，开箱即用。

**Implementation Steps (技术实现步骤):**

**1. 地图初始化 (MapLibre)**

* 引入 MapLibre GL JS 的 CSS 和 JS。
* 初始化地图，使用免费的开源样式（例如 `'https://demotiles.maplibre.org/style.json'`）。
* 设置地图中心点为六盘水附近（例如 `[104.83, 26.59]`），Zoom 设为 16，Pitch 设为 60（以此展示 3D 视角）。

**2. 构建自定义图层 (Custom Layer Interface)**

* 参考 Mapbox/MapLibre 的 `CustomLayerInterface` 规范 。


* 创建一个 `customLayer` 对象：
* `id`: '3d-model'
* `type`: 'custom'
* 
`renderingMode`: '3d' 




* **onAdd(map, gl) 方法**:
* 初始化 Three.js 的 `Camera`, `Scene`, `WebGLRenderer`。
* 
**关键**: 将 `renderer` 绑定到地图的 `gl` 上下文，并设置 `canvas: map.getCanvas()` 。


* 添加环境光 (`AmbientLight`) 和平行光 (`DirectionalLight`)，确保模型可见。
* 使用 `GLTFLoader` 加载示例模型：`https://models.readyplayer.me/64b7a74658867a505e6b4843.glb`。
* 模型加载后，调整其 `scale` 和 `rotation` 使其直立。



**3. 坐标系同步 (Coordinate System Sync) - 核心难点**

* **render(gl, matrix) 方法**:
* 这是实现“数字人站在地图上”的关键 。


* 获取地图投影矩阵：`const m = new THREE.Matrix4().fromArray(matrix);` 。


* 计算模型位置：使用 `maplibregl.MercatorCoordinate.fromLngLat(lngLat, altitude)` 获取模型在墨卡托坐标系中的位置信息 。


* 构建变换矩阵：利用 `matrix` 和模型位置信息，正确设置 `camera.projectionMatrix`，确保 Three.js 的相机与地图相机完全同步 。





**4. 动画循环**

* 在模型加载成功后，创建一个 `AnimationMixer` 并播放第一个动画（通常是 Idle 呼吸动作）。


* 在 `render` 函数中调用 `mixer.update(delta)`。
* 
**重要**: 为了保证动画流畅，必须在 mixer 更新时调用 `map.triggerRepaint()`，否则地图静止时模型动画会卡住 。



**Code Output Requirements:**

* 提供完整的 HTML 代码。
* 代码中必须包含清晰的中文注释，解释“坐标系转换”和“自定义图层”的逻辑。
* 确保代码没有使用任何需要 Key 的服务。
